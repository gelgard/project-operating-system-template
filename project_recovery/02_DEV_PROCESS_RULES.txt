DEV PROCESS RULES

1. Architecture first, then code
2. Work only through small AI tasks
3. Test every task immediately
4. Keep architecture files synchronized
5. Preserve ability to recover context in a new chat
6. Before each new AI task run Fast restore (`обнови контекст`)
7. Run Full restore (`обнови полный контекст`) after architecture sync, merge/stage transition, suspected desync, or long pause
8. Every AI task must include Goal Alignment with Requirement IDs and evidence-based acceptance
